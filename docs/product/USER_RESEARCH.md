# 问渠 AI 命理咨询 - 用户调研与反馈方案

> **目标**: 建立用户反馈机制，持续收集用户需求和产品问题  
> **方法**: 定性研究 + 定量研究结合

---

## 1. 用户研究方法体系

### 1.1 研究方法矩阵

| 研究方法 | 类型 | 样本量 | 周期 | 适用场景 |
|---------|------|--------|------|---------|
| **深度访谈** | 定性 | 5-8人 | 2小时/人 | 需求挖掘、痛点发现 |
| **可用性测试** | 定性 | 3-5人 | 1小时/人 | 交互问题、流程优化 |
| **问卷调查** | 定量 | 50-100人 | 5-10分钟 | 满意度、需求优先级 |
| **用户反馈** | 定性 | 不限 | 即时 | 问题收集、建议采纳 |
| **行为分析** | 定量 | 全量 | 持续 | 使用习惯、转化漏斗 |

### 1.2 研究流程

```
明确研究目标 → 选择方法 → 设计提纲 → 招募用户 → 执行研究 → 分析洞察 → 落地改进
```

## 2. 深度访谈方案

### 2.1 访谈目标

- 了解用户使用命理咨询的真实动机
- 发现当前产品的体验痛点
- 收集新功能需求
- 验证产品价值主张

### 2.2 用户招募

**筛选条件**:
- 应届毕业生或职场新人（22-28岁）
- 有过求职焦虑经历
- 对AI技术不排斥
- 愿意接受1-2小时访谈

**招募渠道**:
- 朋友圈、微信群
- 校园社群
- 脉脉、LinkedIn

### 2.3 访谈提纲

#### 开场 (5分钟)
- 自我介绍，说明目的
- 签署知情同意书
- 建立轻松氛围

#### 背景了解 (10分钟)
1. 请简单介绍一下你自己？
2. 你平时会关注星座、八字这些内容吗？
3. 最近一次感到求职焦虑是什么时候？能描述一下吗？

#### 核心问题 (30分钟)
4. **动机探索**: 如果你在做重要决定前感到迷茫，你会怎么办？
5. **当前方案**: 你有尝试过命理咨询吗？体验如何？
6. **痛点挖掘**: 你觉得现有方案有哪些不足？
7. **AI接受度**: 你怎么看AI做命理咨询？有什么顾虑？

#### 产品测试 (30分钟)
8. **第一印象**: 看到这个产品，你觉得是做什么的？
9. **功能探索**: 试着完成一次测算，边操作边说出你的想法。
10. **体验反馈**: 刚才使用过程中，有哪些地方让你困惑或不满意？
11. **价值感知**: 如果这个问题得到满意的答案，对你有什么帮助？

#### 结束 (10分钟)
12. 如果给这个产品提一个建议，会是什么？
13. 你会向朋友推荐这个产品吗？为什么？

## 3. 可用性测试方案

### 3.1 测试目标

- 发现交互流程中的卡点
- 验证设计方案的可理解性
- 测量任务完成时间和成功率

### 3.2 测试任务设计

| 任务 | 目标 | 成功标准 |
|------|------|---------|
| 任务1: 完成一次八字测算 | 测试核心流程 | 2分钟内完成 |
| 任务2: 切换不同咨询方向 | 测试导航理解 | 正确切换并理解差异 |
| 任务3: 分享结果给朋友 | 测试分享功能 | 找到分享入口 |
| 任务4: 查看历史记录 | 测试历史功能 | 正确找到入口 |

### 3.3 出声思考法

**指导语**:
> "请您在使用产品时，把您看到的、想到的、感受到的都说出来。比如'我现在要点这个按钮'、'这里我不太明白'、'这个动画很好看'等等。"

### 3.4 测试记录表

| 任务 | 用户1 | 用户2 | 用户3 | 平均 | 问题 |
|------|-------|-------|-------|------|------|
| 八字测算 | 3min ✅ | 5min ⚠️ | 2min ✅ | 3.3min | U1：时辰选择困惑 |
| 切换方向 | ✅ | ✅ | ⚠️ | - | U3：没发现切换入口 |
| 分享结果 | ⚠️ | ❌ | ✅ | - | U2：找不到分享按钮 |

## 4. 问卷调查方案

### 4.1 调研目标

- 量化用户满意度
- 了解功能使用频率
- 收集用户画像信息
- 优先级排序

### 4.2 问卷设计

#### 第1部分: 用户画像 (3题)

1. 您的年龄段？
   - 18-22岁 / 23-25岁 / 26-28岁 / 29岁以上

2. 您目前的身份？
   - 在校学生 / 应届毕业生 / 职场新人(1-3年) / 职场老手

3. 您使用过命理咨询吗？
   - 经常使用 / 偶尔使用 / 很少使用 / 从未使用

#### 第2部分: 产品体验 (5题)

4. 您使用问渠的频率？
   - 每天 / 每周几次 / 每月几次 / 只用了一次

5. 您对AI回答的满意度？(1-5分)
   - ⭐⭐⭐⭐⭐

6. 您认为AI回答最准确的功能是？
   - 八字分析 / 六爻占卜 / 合盘匹配

7. 您是否会推荐问渠给朋友？(NPS)
   - 0-10分

8. 您使用问渠的主要场景是？
   - 求职决策 / 感情困惑 / 日常好奇 / 其他____

#### 第3部分: 改进建议 (2题)

9. 您最希望增加的功能是？
   - 历史记录 / 多轮对话 / 语音输入 / 社区交流 / 其他____

10. 您对问渠的其他建议？
    - [开放题]

### 4.3 问卷发放

**发放渠道**:
- 产品内弹窗邀请
- 微信群、朋友圈
- 用户完成测算后弹出

**激励机制**:
- 填写问卷获得"免费测算券"
- 抽奖送小礼品

## 6. NPS调研方案

### 6.1 NPS问题

> "您有多大可能将问渠推荐给朋友或同事？(0-10分)"

### 6.2 评分标准

| 分数 | 用户类型 | 说明 |
|------|---------|------|
| 9-10分 | 推荐者(Promoters) | 忠实用户，会主动推荐 |
| 7-8分 | 被动者(Passives) | 满意但不会主动推荐 |
| 0-6分 | 贬损者(Detractors) | 不满意，可能负面传播 |

### 6.3 NPS计算公式

```
NPS = 推荐者% - 贬损者%

示例:
- 推荐者: 50人 (50%)
- 被动者: 30人 (30%)
- 贬损者: 20人 (20%)

NPS = 50% - 20% = 30
```

### 6.4 跟进问题

根据评分询问原因：

**推荐者 (9-10分)**:
> "感谢您的高分评价！请问问渠最打动您的地方是什么？"

**被动者 (7-8分)**:
> "谢谢您的评分。如果我们做一点改进就能让您给10分，那会是什么？"

**贬损者 (0-6分)**:
> "非常抱歉没能让您满意。能告诉我们哪里做得不好吗？"

## 7. 研究计划与排期

### 7.1 研究节奏

| 时间 | 研究活动 | 产出 |
|------|---------|------|
| **每月第1周** | 深度访谈(3-5人) | 用户洞察报告 |
| **每月第2周** | 可用性测试(3-5人) | 体验问题清单 |
| **每季度** | 问卷调研(50+人) | 满意度报告 |
| **持续** | 用户反馈收集 | 需求池更新 |

### 7.2 首次研究计划

**Week 1-2: 深度访谈**
- 招募5位目标用户
- 完成访谈并整理洞察
- 产出：用户画像 + 痛点清单

**Week 3: 可用性测试**
- 邀请3位新用户
- 测试核心功能流程
- 产出：体验问题清单

**Week 4: 问卷调研**
- 设计并发布问卷
- 收集50+份有效问卷
- 产出：满意度基线数据

## 8. 研究成果落地

### 8.1 洞察到产品的转化

```
用户反馈 → 问题归类 → 优先级排序 → 产品设计 → 开发实现 → 效果验证
```

### 8.2 用户反馈处理时效

| 反馈类型 | 响应时间 | 处理时间 |
|---------|---------|---------|
| 紧急Bug | 2小时内 | 24小时内 |
| 功能建议 | 24小时内 | 评估后排期 |
| 体验问题 | 24小时内 | 下次迭代 |
| 一般咨询 | 48小时内 | 及时回复 |

### 8.3 用户感谢机制

- 反馈被采纳的用户，在产品更新日志中致谢
- 定期抽取提供有价值反馈的用户赠送礼品
- 建立"产品体验官"机制，邀请活跃用户参与内测
